## Task 3: Update CI test job to generate and upload coverage

**File:** `/Users/dmytro.lupiak/GitHub/claude-session-dashboard/.github/workflows/ci.yml`
**Complexity:** Low (5 min)

Two changes in the `test` job:

### Change 1: Replace test command (line 56)

```diff
      - name: Run tests
-       run: npm run test
+       run: npx vitest run --coverage
```

### Change 2: Add Codecov upload step after the test step (after line 56)

```yaml
      - name: Upload coverage to Codecov
        if: github.event_name == 'push' || github.event_name == 'pull_request'
        uses: codecov/codecov-action@671740ac38dd9b0130fbe1cec585b89eea48d3de # v5.5.2
        with:
          files: ./apps/web/coverage/lcov.info
          fail_ci_if_error: false
```

### Full test job after changes

```yaml
  test:
    name: Unit Tests
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: ./apps/web
    steps:
      - name: Checkout code
        uses: actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd # v6

      - name: Setup Node.js
        uses: actions/setup-node@6044e13b5dc448c55e2357c09f80417699197238 # v6
        with:
          node-version: '22'
          cache: 'npm'
          cache-dependency-path: ./apps/web/package-lock.json

      - name: Install dependencies
        run: npm ci

      - name: Run tests
        run: npx vitest run --coverage

      - name: Upload coverage to Codecov
        if: github.event_name == 'push' || github.event_name == 'pull_request'
        uses: codecov/codecov-action@671740ac38dd9b0130fbe1cec585b89eea48d3de # v5.5.2
        with:
          files: ./apps/web/coverage/lcov.info
          fail_ci_if_error: false
```

### Notes

- `fail_ci_if_error: false` prevents CI from failing if Codecov is temporarily unavailable.
- The `files` path is relative to the repo root (not `working-directory`), so it must include `./apps/web/`.
- The action is pinned to the full commit SHA matching v5.5.2, consistent with existing action pinning convention.
- For public repos, Codecov does not require a `CODECOV_TOKEN` secret (tokenless uploads work via GitHub OIDC). If uploads fail, add the token from codecov.io as a repo secret.

### Verification

- Push branch, open PR, confirm the "Upload coverage to Codecov" step runs green.
- Check codecov.io/gh/dlupiak/claude-session-dashboard for the uploaded report.

## External Setup Required

- **Codecov:** The repo must be activated on [codecov.io](https://codecov.io). For public GitHub repos, this is free and can use tokenless uploads. If tokenless upload fails, generate a `CODECOV_TOKEN` from the Codecov dashboard and add it as a GitHub repo secret.
- **Socket.dev:** No setup needed -- the badge is a static shields.io link pointing to the Socket.dev package page. The package is already published on npm, so the Socket page exists automatically.
